# Component Consistency

## Overview

Make repeated UI patterns feel like they belong to one system.

## Priorities

Work in this order:

1. Buttons and CTAs
2. Cards
3. Section headers
4. Repeated layout patterns
5. Hover and border treatment

## Core Rules

- Make similar components share the same visual logic.
- Align spacing, padding, border weight, and radius decisions.
- Keep typography treatment consistent across repeated elements.
- Make hover behavior coherent across the site.
- Let high-value components stand out through hierarchy, not inconsistency.

## Watch For

- Buttons with mismatched padding or weight
- Cards with uneven density or image treatment
- Sections with inconsistent header spacing
- Mixed border, shadow, or background logic
- One-off styles that break the system

## Implementation Guidance

- Standardize before embellishing.
- Prefer reusable adjustments over isolated one-off fixes.
- Keep the system compact and predictable.

## Output Expectations

When using this skill:

1. Audit repeated UI patterns
2. Normalize styling decisions
3. Leave the interface more cohesive and balanced
